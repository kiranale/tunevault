📖 How to Setup

🌐 Proxy Setup Guide

Run oracle-proxy.py on your Oracle server (Python 3.6+). It listens on localhost:3100 — your outbound HTTPS proxy forwards requests to it.

1. Download & Install
# On your Oracle server (Oracle Linux 7+ / RHEL) wget https://tunevault.app/downloads/oracle-proxy.py wget https://tunevault.app/downloads/oracle-proxy-install.sh chmod +x oracle-proxy-install.sh sudo ./oracle-proxy-install.sh
2. Create proxy.env

The installer creates /etc/oracle-proxy/proxy.env. Fill it with your values (or copy from the form on the left):

TUNEVAULT_API_KEY=<your-generated-key> TUNEVAULT_PROXY_URL=https://<your-proxy-host>/proxy ORACLE_HOST=localhost ORACLE_PORT=1521 ORACLE_SERVICE=<your-service-name>
3. Verify it works
# From any machine: curl https://<your-proxy-host>/health # Should return: {"status":"healthy","version":"..."}

TuneVault will call https://<your-proxy-host>/proxy — this is the endpoint your HTTPS proxy must forward to http://localhost:3100.

Required Oracle Grants

Create a dedicated read-only Oracle user, then grant it access to performance views.

Step 1: Create the user
-- Create the TuneVault read-only user -- Replace <your_password> with your chosen password CREATE USER tunevault_reader IDENTIFIED BY <your_password> DEFAULT TABLESPACE users TEMPORARY TABLESPACE temp; -- Grant login privilege GRANT CREATE SESSION TO tunevault_reader;
Step 2: Grant read access (choose one option)
Option 1: Simple (full catalog read)
GRANT SELECT_CATALOG_ROLE TO tunevault_reader;
Option 2: Minimum grants
-- Instance & database info GRANT SELECT ON V_$INSTANCE TO tunevault_reader; GRANT SELECT ON V_$DATABASE TO tunevault_reader; GRANT SELECT ON V_$PARAMETER TO tunevault_reader; GRANT SELECT ON V_$VERSION TO tunevault_reader; -- Performance data GRANT SELECT ON V_$SQL TO tunevault_reader; GRANT SELECT ON V_$SYSTEM_EVENT TO tunevault_reader; GRANT SELECT ON V_$SYSSTAT TO tunevault_reader; GRANT SELECT ON V_$LIBRARYCACHE TO tunevault_reader; GRANT SELECT ON V_$ROWCACHE TO tunevault_reader; GRANT SELECT ON V_$OSSTAT TO tunevault_reader; -- Memory GRANT SELECT ON V_$SGA TO tunevault_reader; GRANT SELECT ON V_$SGASTAT TO tunevault_reader; GRANT SELECT ON V_$PGASTAT TO tunevault_reader; GRANT SELECT ON V_$SQL_WORKAREA_HISTOGRAM TO tunevault_reader; -- Tablespace & storage GRANT SELECT ON DBA_TABLESPACE_USAGE_METRICS TO tunevault_reader; GRANT SELECT ON DBA_TABLESPACES TO tunevault_reader; GRANT SELECT ON DBA_DATA_FILES TO tunevault_reader; GRANT SELECT ON DBA_FREE_SPACE TO tunevault_reader; -- Index analysis GRANT SELECT ON DBA_INDEXES TO tunevault_reader; GRANT SELECT ON DBA_IND_STATISTICS TO tunevault_reader;

🌐 Quick Access URLs

Save the EBS login page URL and WebLogic Admin Console URL for this connection. Open EBS and WebLogic buttons will appear on the connection — clicking them routes the request through your proxy agent so you reach internal URLs without VPN.

ⓘ Set APPS_URL_WHITELIST=host:port,host:port in your proxy.env so the proxy accepts these targets. Example:
APPS_URL_WHITELIST=ebs-app.local:8000,ebs-app.local:7001

⚠️ Regenerate API Key?

This will generate a new API key. Your current key will stop working immediately.

You must update TUNEVAULT_API_KEY in your proxy.env file and restart the proxy before TuneVault can connect again.
